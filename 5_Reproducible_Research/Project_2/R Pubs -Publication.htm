Okay, I finally got it to work, here's what I did:
search your harddrive for the files .rprofile and .site
Open them with word or another text editor
put the code in as suggested,:
options(rpubs.upload.method = "internal")

 I have it as 1st line, don't know if that is required or not, but that's where it is in my files

Then copy these 2 (profile and site) files into the folder that you have you markdown (".Rmd") file in
Reboot your Rstudio, reknit again.
Then follow the instructions from the assignment (i.e. publish, and further)

David HoodCommunity TA· 12 days ago 

Getting it in as a .Rprofile file is, I understand, the only reliable way because of order of operations. So before trying any really complicated work arounds, let's check some things:

  • On your computer system, could you be using a program to create the file that adds a suffix (like .txt) or rejects the dot at the start. Make sure the file is simply called .Rprofile
  • Depending on computer system, you might be able to run the options(rpubs.upload.method = "internal") command directly in the console before trying to publish the file, but as I understand it that is not ideal.
  • Modifying the general .RProfile should be possible if you have admin access to the computer, but as it is a change that would effect other users of the computer it is not something they let you do casually.

Jack Francis· 2 days ago 
There's a problem in Windows when creating files beginning with a dot; Windows takes a single dot as meaning the current directory.  Here is one solution if you have Git installed.
Go to Start and type Git to get the Git Bash command window.
Go to the directory containing your .rmd file.  Type pwd to get the directory you're in, then use a bunch of CD commands to get to the proper directory.  Use the Unix forward slash rather than the Windows back slash.
Enter 'touch .Rprofile' to create the file.
Use Windows Explorer to locate the file, right click on it and edit it with your favorite editor.  Enter the line:
options(rpubs.upload.method = "internal")

I've gotten this to work as well, though not as previously suggested. So if you are still having trouble here's something else to try.  In the .Rprofile and .Rprofile.site in their original locations, put in the options line.  You can tell after you start RStudio if this has worked by issuing the command options(). You should see an option for rpubs.upload.method that has the value 'internal' in the long list returned. Apparently, RStudio only looks at the options list at startup and not after, which is why issuing options on the command line doesn't help.  Good luck.

I am not a expert on this however this one worked for me, even though I got several errors initially.

1. Go to C:/Program File:/R/Library/base/R
2. Copy Rprofile File
3. Paste in your working directory where your .Rmd file is saved
4. Open using Notepad
5. Do not delete anything. At the top copy and paste following line options(rpubs.upload.method = "internal")
6. Save it. Close R if already open
7. Reopen Rmd file. Knit it. And Publish it.
8. Sometimes restarting the system does the trick.
